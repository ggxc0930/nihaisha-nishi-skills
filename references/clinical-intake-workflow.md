# 用户身体情况与方药对标工作流

> 本模块用于把用户描述的身体情况、疾病名、方子、舌苔/脉象信息，转成倪海厦课程资料检索和学习型对标分析。它不是医疗诊断、处方或剂量建议。

## 适用输入

- 身体情况：怕冷、怕风、发热、汗出、口渴、口苦、睡眠、大小便、疼痛部位、咳嗽痰饮、胸腹症状、妇科情况等。
- 疾病名：用户提供的现代医学诊断名、检查名、病程和既往治疗。
- 中药方子：方名、组成、剂量、服用后反应、来源。
- 舌苔图片或文字描述：舌色、苔色、厚薄、润燥、齿痕、裂纹、瘀点等。
- 脉象描述：浮沉、迟数、虚实、弦滑细涩、左右寸关尺差异等。

## 必问信息

若用户只给一个症状或方子，先补问或标注缺失：

| 类别 | 关键问题 |
| --- | --- |
| 表证 | 怕冷还是怕风？有汗还是无汗？发热程度？头项强痛？ |
| 里证 | 大便、小便、口渴、腹满腹痛、食欲、恶心呕吐。 |
| 寒热 | 手脚冷、烦躁、口干口苦、喜冷喜热、舌苔颜色。 |
| 虚实 | 体力、声音、出汗、心悸、气短、疼痛按压喜恶。 |
| 睡眠情志 | 入睡、早醒、多梦、烦躁、惊悸。 |
| 病程风险 | 年龄、孕产、儿童、基础病、用药、检查结果、急症信号。 |

## 检索顺序

1. 先运行跨资料检索：

```bash
python scripts/search_references.py <症状 疾病名 方名 舌苔 脉象>
```

2. 若用户问方子，打开 `formula-patterns.md`，再查 `six-channel.md` 和相关课程模块。
3. 若用户问疾病名或案例，打开 `clinical-cases.md`、`jingui.md`、`bagang.md`，必要时查 `diagnostic-logs.md`。
4. 若用户给舌苔/脉象，优先查 `huangdi.md`、`bagang.md`、`zhongjing-xinfa.md`、`clinical-cases.md` 中的诊断线索。
5. 若用户给图片，先描述可见特征和图像限制，再把可见特征转成文字检索词；不要仅凭图片下诊断。
6. 需要板书或证据时运行：

```bash
python scripts/search_screenshots.py <关键方名 症状 病机 穴位>
```

## 输出结构

按以下结构回答：

1. **边界**：说明这是课程资料学习型对标，不是诊断或处方。
2. **已知信息**：复述用户提供的症状、疾病名、方子、舌苔/脉象。
3. **缺失信息**：列出影响辨别的关键问题。
4. **课程对标方向**：列 2-4 个可能相关的课程方证/病机方向，并说明支持点和不支持点。
5. **方子分析**：若有方子，说明它在课程里的常见方证、风险点、与相近方的分水岭；不改方、不给剂量。
6. **舌脉线索**：只把舌脉作为辅助线索，说明图像/文字描述的不确定性。
7. **证据索引**：列出 references 文件、课次、截图路径或检索命中。
8. **风险提醒**：出现急症、孕产儿童、肿瘤、胸痛、意识改变、严重脱水、附子类/峻下攻逐方等，建议及时找合格医生或急诊。

## 禁止输出

- 不给“你就是某某证”的确定诊断。
- 不给个人化处方、剂量、加减、煎服法。
- 不指导用户自行停药、换药或替代医生治疗。
- 不把舌苔图片当成唯一依据。
